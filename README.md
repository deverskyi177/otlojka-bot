# Sialens

Telegram-бот на aiogram 3 для модерации групп, игр, аналитики и триггеров, с VIP-подпиской через Telegram Stars. Деплой на Railway.

## ⚠️ Важное техническое уточнение

Бот умеет работать в **двух режимах**, и они не заменяют друг друга:

### 1. Группы (классический режим)
Добавьте бота в группу администратором с правами «удаление сообщений» и «блокировка пользователей» — команды `.mute`, `.ban`, `.kick`, `.clean`, `.warn` и автофильтры работают там как обычный модератор.

### 2. Личные чаты через «Автоматизацию чатов» (Telegram Business)
Как на ваших скриншотах: в Telegram (Настройки → Telegram Business → Автоматизация чатов) вы подключаете бота к своему аккаунту. После этого Telegram присылает боту сообщения из ваших личных переписок как специальные апдейты `business_message`.

Реализовано так: если **вы сами** (владелец аккаунта) пишете собеседнику `.mute`, `.ban`, `.warn`, `.unmute`, `.unban` или `.clean` прямо в переписке с ним — бот перехватывает это сообщение, удаляет его (чтобы собеседник не увидел саму команду) и применяет действие к этому конкретному человеку:

- **`.mute [1ч/6ч/12ч/24ч/навсегда]`** — бот отправляет собеседнику «Замолчи.» и с этого момента молча удаляет все его новые сообщения в этом чате.
- **`.ban`** — бот молча удаляет все дальнейшие сообщения от этого человека, без ответа (имитация блокировки).
- **`.warn`** — увеличивает счётчик предупреждений; при 3 предупреждениях статус автоматически становится «ban».
- **`.unmute` / `.unban`** — снимает ограничение.
- **`.clean [N]`** — пытается удалить последние N сообщений в этой переписке.

**Важные ограничения (не ошибка бота, а ограничение самого Bot API):**
- У ботов нет метода «заблокировать контакт» — это действие доступно только вручную в приложении Telegram. `.ban` не мешает человеку писать, а лишь гарантирует, что вы никогда не увидите его сообщения (они удаляются автоматически).
- Удаление чужих сообщений через Business-подключение работает, только если при подключении бота вы включили ему соответствующее разрешение («Разрешения для бота» на скриншоте) и только для сообщений, которые Telegram ещё разрешает удалять (как правило, недавние).
- Всё это работает только внутри чатов, которые вы явно разрешили боту в разделе «Все личные чаты, кроме…» / «Только выбранные чаты».
- Требуется Telegram Premium (Business-аккаунт нужен для функции «Автоматизация чатов»).

## Функционал

- **Модерация:** `.mute`, `.unmute`, `.warn`, `.ban`, `.unban`, `.kick`, `.clean`, `.wbl` (чёрный список слов), `.wsag` (блокировка стикеров/гифок)
- **Автофильтры:** капс >70%, ссылки, повторяющиеся сообщения, флуд (4+ сообщений за 5 сек → авто-мут)
- **Автобан:** 3 предупреждения → бан на 24ч
- **Игры:** `.games`, `.rps`, `.anim`, `.revo`, `.guess`
- **Аналитика (VIP):** `.stats`, `.top`, `.nightowl`, `.peak`
- **Триггеры (VIP):** `.addtrigger`, `.removetrigger`, `.trigger`
- **Опросы:** `.poll "вопрос" "вар1" "вар2"`
- **Напоминания:** `.remind "текст" 10м`
- **Смена раскладки:** `.sw <текст>`
- **Приветствие/прощание** при входе/выходе участников
- **Роли:** новичок (0-50 сообщ.) → проверенный (50+) → VIP (подписка) → админ
- **Подписка:** триал (по умолчанию 3 дня) + оплата звёздами Telegram (`/buy`)
- **Админ-панель:** `/admin` с инлайн-кнопками
- **Команды админа:** `/givepremium`, `/removepremium`, `/addadmin`, `/removeadmin`, `/banuser`, `/unbanuser`, `/setprice`, `/settrial`, `/resetwarns`, `/resetstats`
- SQLite база создаётся автоматически (`data/sialens.db`)
- Логи пишутся в `logs/bot.log`

## Установка локально

```bash
git clone <ваш-репозиторий>
cd sialens
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
```

Заполните `.env`:

```
BOT_TOKEN=токен_от_BotFather
ADMIN_IDS=123456789,987654321
```

Запуск:

```bash
python bot.py
```

## Настройка бота в группе

1. Создайте бота через [@BotFather](https://t.me/BotFather), получите токен.
2. Добавьте бота в свою группу.
3. Назначьте бота администратором с правами: удаление сообщений, блокировка пользователей, закрепление сообщений (для авто-модерации).
4. Отправьте `/start`, затем `/admin`, чтобы открыть панель управления (доступна только ID из `ADMIN_IDS` или добавленным через `/addadmin`).

## Деплой на Railway

1. Зайдите на [railway.app](https://railway.app) и создайте новый проект → **Deploy from GitHub repo**, выбрав репозиторий с этим кодом.
2. В разделе **Variables** добавьте:
   - `BOT_TOKEN`
   - `ADMIN_IDS`
3. Railway автоматически определит `Procfile` и запустит процесс `worker: python bot.py`.
4. Убедитесь, что в настройках сервиса выбран тип процесса **Worker** (не Web), так как бот работает через long polling, а не HTTP.
5. Для сохранения базы данных между деплоями подключите **Railway Volume** и примонтируйте его в папку `data/`, иначе статистика/подписки будут сбрасываться при каждом передеплое.

## Telegram Stars (подписка)

Оплата подписки использует нативные звёзды Telegram (`currency="XTR"`), поэтому платёжный провайдер настраивать не нужно — это работает "из коробки" для любого бота.
